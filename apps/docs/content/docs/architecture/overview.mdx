Every app is a thin presentation layer over one shared package, `@zk-freighter/core`, which owns
identity, vault, proving, networks, and transactions. If a flow works on web it works on the
extension and mobile, because they run the same code.

## The monorepo

| Path | What it is |
|---|---|
| `apps/web` | React wallet app — the primary surface |
| `apps/extension` | WXT MV3 browser extension (popup + side panel, offscreen prover) |
| `apps/mobile` | Capacitor app (Android/iOS) reusing the same core |
| `apps/landing` | Product site |
| `apps/bootnode` | Warmed pool-event indexer + narrow RPC (Postgres) |
| `apps/funding-api` | Hosted testnet faucet (XLM + USDC, budgets and cooldowns) |
| `packages/core` | All wallet, network, proof, and transaction logic |
| `packages/ui` | Shared design system |
| `circuits/`, `contracts/` | Noir circuits + Soroban contract for Confidential Token mode |

## Networks are configuration

Testnet ↔ mainnet is a config toggle, not a code change: RPC/Horizon endpoints, pool contract
IDs, asset contracts, and bridge routes all come from one network config. The UI always shows
which network you're on.

## Two privacy modes, two proving stacks

| | Shielded Pools (flagship) | Confidential Tokens (preview) |
|---|---|---|
| Hides | balances, amounts, counterparties | amounts and balances only |
| Proving | Circom / Groth16, Rust→WASM | Noir + UltraHonk (bb.js) |
| Networks | testnet + mainnet | testnet only |

Read more in [Proving](/docs/architecture/proving) and
[Confidential mode](/docs/apps/confidential).
